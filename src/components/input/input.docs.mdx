# Input

## Overview

`Input` is a simple styled wrapper around the html5 `input` element.

### Customisation

- Input allows customization through top-level props.
- The props map to the `getSprinkles` utility for composing additional styles.

<Canvas sourceState="shown" of={stories.CustomisationMargin} />

### "slot" props

To allow more fine-grained control over the bundling of dependencies, the
`Input` component does not include any icon library. Instead, it accepts a
`slotLeft` and `slotRight` prop, which can be used to render any React node
inside the input, on the corresponding side.

### Size

The `Input` has three size variants: `"sm"`, `"md"`, and `"lg"`.

- `"sm"` is for small secondary content, like nav bars or sidebars
- `"md"` is the default size, and meets Apple's minimum tap target size
  requirements
- `"lg"` is for edge cases, like marketing CTAs

### State

The `Input` component has state-specific styles that respective the following
selectors

- :hover
- :focus-visible
- \[disabled] or \[aria-disabled="true"]
