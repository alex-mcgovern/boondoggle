# Button

<Canvas of={stories.Default} />

`Button` is a simple wrapper around the html5 `button` element, with support for
style variants, quick customisation via `getSprinkles` and an `as` prop that
enables polymorphism, (e.g. if you would like to return an `<a>` element, or a
custom `Link` component instead of the normal `<button>` element.).

### `as` (polymorphism)

<ArgTypes of={Button} include={["as"]} />

`Button` is a polymorphic component that accepts an `as` prop to change the
underlying HTML element. You can pass the name of a generic HTML element, or you
can pass a React component. By default it will return a `<button>` element.

A common use case is to style a link as a button. This component is designed to
be compatible with a range of React frameworks and routing solutions. In these
examples, we are using an `<a>` tag for a standard anchor link, and a mocked
custom Link component, similar to the one provided by frameworks like
[Reach Router](https://reach.tech/router/api/Link),
[NextJS](https://nextjs.org/docs/api-reference/next/link) or
[Gatsby](https://www.gatsbyjs.com/docs/reference/built-in-components/gatsby-link/).

### "slot" props

<ArgTypes of={Button} include={["slotLeft", "slotRight"]} />

To allow more fine-grained control over the bundling of dependencies, the
`Button` component does not include any icon library. Instead, it accepts a
`slotLeft` and `slotRight` prop, which can be used to render any React node
inside the button, on the corresponding side.
